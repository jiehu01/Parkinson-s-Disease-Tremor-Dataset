We compiled multiple Parkinson‚Äôs disease (PD) tremor datasets, each collected using IMU sensors. Tremor severity in these datasets is labeled on a scale from 0 to 3, where 0 indicates no tremor, and 1‚Äì3 represent increasing levels of tremor severity. Further details can be found in the paper Time Series Adaptation Network for Sensor-based Human Activity Recognition.

1. TIM-Tremor Dataset [1]

Collected postural and resting tremor data from 55 patients using accelerometers attached to the back of the hand, along with video recordings. Patients performed tasks like counting, tapping, and piano playing to trigger tremor. Sampling frequency: 1000 Hz. Labels linearly mapped (label_new = label/2).

2. PdAssist Dataset [2]

Collected via smartphone app with guided tasks (arm hanging, balance, gait, postural tremor, resting tremor). Used accelerometer data from resting tremor tests of both hands. Sampling frequency: 50 Hz. Contains data from hundreds of patients.

3. IMU-Wild Dataset [3]

Data collected outdoors from 45 subjects (31 PD, 14 healthy) during tremor episodes using accelerometer and gyroscope sensors. Used tremor data from ‚Äúphone-calling‚Äù tasks. Sampling frequency: 50 Hz.

4. PD-BioStamp Dataset [4]

Collected with 5 wearable sensors on chest and limbs from 34 PD patients during medication ‚Äúon/off‚Äù periods. Used accelerometer data from two upper-limb sensors. Sampling frequency: 50 Hz.

Data Organization

Only accelerometer data are used. Data are organized as <X.npy, Y.npy> pairs:

  • X.npy: shape [num_samples, 128, 3], stores sensor data
  • Y.npy: shape [num_samples], stores labels

Example: X.npy = [5,128,3], Y.npy = [5] ‚Üí 5 samples per Raw Segment.

Data Preprocessing

- Sampling frequency unified to 50 Hz (linear interpolation).
- Sliding window: length 128 (2.56s), non-overlapping.
- Variable-length raw segments divided into fixed-size samples.
